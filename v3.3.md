=> update lại về phần lưu trữ file input và file logs: cả 2 sẽ bắt đầu với đường dẫn là: resources/users/{email_user}/ sau đó chia ra làm 2 là input và logs
- input: chứa các file input của user
- logs: chứa các file logs của user cho từng pipeline để có thể dễ dàng xem lại và quản lý. file logs này sẽ được lưu trữ theo ngày, ví dụ như các pipeline chạy trong ngày hôm nay sẽ được lưu vào file logs/2026-04-02 và các pipeline chạy trong ngày hôm qua sẽ được lưu vào file logs/2026-04-01 và cứ thế tiếp tục. 
=> các file json như sessions, quota có đường dẫn là resources/system/

- quota.json
