step1: input sẽ là các file ảnh và address. Tạo ra 1 uuid random làm unique_str, kết quả response trả về sẽ là presignedUrls có danh sách url và filename. Ngoài ra lưu lại ảnh vào 1 folder có tên format là {unique_str}/{address}/input
step2: sử dụng output ở step 1 là list url và filename là input. lần lượt sử dụng các url từ presignedUrls để upload file lên s3 với file sử dụng là binary mà input từ step 1 mỗi file tưng ứng với 1 url trong presignedUrls, sau khi upload xong thì response trả về với status là 200 với request theo method là put vào options. output sẽ là true và false tùy theo response trả về
step3: ở step 3 nếu output ở step 2 là true thì sử dụng unique_str để gửi request và output sẽ là true hoặc false dựa trên status. Tuy nhiên tôi muốn kiểm tra response có thay đổi gì không thì hiện tại response trả về là 1 json như sau:

để thêm 1 trường hợp có thể bị false đó chính là trong nội dung trả về trong response không chứa từ khóa: 'successfully' thì sẽ là false còn không thì dựa trên status

step4: step 3 trả về true thì sẽ tiếp tục đến step 4. unique_str từ step 1 sẽ là input để gửi request đến url vs thông tin email, firstname, lastname, address sẽ là được nhập từ input addres trùng với address ở step1

Step 7: 
tải các ảnh dựa trên danh sách url đã loại bỏ param từ step 6. và lưu vào 1 folder path là {unique_str}/{address}/output. tải ảnh xong thì hiển thị log "Info - step7: Tải ảnh thành công" còn nếu không tải được thì hiển thị log "Error - step7: Tải ảnh thất bại" và retry lại sau 15s và sau mỗi lần như vậy sẽ nhân cho 3/2 thời gian đợi trong 10 lần nếu retry thì hiển thị số lần retry và thời gian đợi (quy ra xấp sỉ phút)
Nếu thành công thì update 1 file json nếu đã tồn tại và insert vào json nếu chưa tồn tại và file json có schema như sau:
email (string), unique_str (array<string>), count (int), limit_count (int), limit_file (int)
nếu email đã tồn tại thì thêm unique_str vào mảng unique_str chỉ khi unique_str chưa tồn tại trong mảng unique_str, về count sẽ được cộng dồn vào với mỗi lần download file thành công, (ví dụ như lần đầu tiên count = 0, sau khi download 10 ảnh thì count = 10, lần sau download 5 ảnh thì count = 15). Trước khi download ảnh thì sẽ đếm số file trong danh sách url đã loại bỏ param từ step 6 nếu <= limit_file thì tiếp tục download, nếu > limit_file thì hiển thị log "Error - step7: Số file vượt quá limit file" và dừng lại. ngoài ra check thử xem limit count - (count+số file download) > 0 thì tiếp tục download, nếu <= 0 thì hiển thị log "Error - step7: Số file vượt quá limited, bạn có thể tải nó trong phần xem ảnh đã xử lý" và dừng lại.


Log được format theo dạng;
<LEVEL: STEP_NUMBER: MESSAGE>

LEVEL: INFO (Hiển thị thông tin từng bước), ERROR (Báo lỗi), DEBUG (Hiển thị thông tin debug mà không trả về kết quả)
STEP_NUMBER: 1, 2, 3, 4, 5, 6, 7
MESSAGE: message
